# Исправление ошибки "request aborted"

## Проблема
Ошибка `BadRequestError: request aborted` возникает когда клиент закрывает соединение до завершения обработки запроса сервером.

## Причины:
1. Клиент (браузер) закрывает соединение (таймаут, пользователь закрыл страницу)
2. Nginx или gateway закрывает соединение из-за таймаута
3. Нормальное поведение при прерванных запросах

## Что было исправлено:

### Auth Service (services/auth-service/server.js)
- ✅ Добавлена обработка ошибок body parser
- ✅ Добавлены обработчики для игнорирования ошибок "request aborted"
- ✅ Добавлены обработчики на уровне процесса для uncaughtException и unhandledRejection

## Важно:

Ошибка "request aborted" - это **нормальное поведение** в следующих случаях:
- Пользователь закрыл страницу во время отправки запроса
- Таймаут на стороне клиента
- Проблемы с сетью

Сервер должен **игнорировать** эти ошибки, так как клиент уже закрыл соединение и ответ не нужен.

## После исправлений:

Перезапустите auth-service:
```bash
docker-compose -f docker-compose.microservices.yml restart auth-service
```

Эти ошибки больше не будут выводиться в логи, но это не критично - они не влияют на работу приложения.
